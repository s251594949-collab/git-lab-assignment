# IMC22403 Assignment 1 – Git & GitHub Workflow

## What I Learned
Through this assignment, I learned how to use Git and GitHub to manage a software project efficiently. I learned how to initialize a repository, create commits, track file changes, create branches, merge branches, and resolve merge conflicts. I also learned how Pull Requests work in collaborative development.

## Challenges Faced
One of the main challenges was resolving merge conflicts because both branches modified the same line in a file. At first, it was confusing to understand the conflict markers shown by Git. However, after manually editing the file and completing the merge, I understood how conflict resolution works.

Another challenge was remembering the correct Git commands and understanding the difference between local and remote repositories.

## How Git Helps in Real Projects
Git helps developers manage and organize project changes effectively. It allows multiple developers to work on different features without affecting the main project directly. Branching and Pull Requests improve teamwork and code review processes. Git also keeps a history of changes, making it easier to track mistakes and restore previous versions if needed.

## Reflection
Version control improves software development quality by helping developers track changes, collaborate safely, and avoid losing important work. Git supports teamwork because developers can work on separate branches and merge their work later. It also helps teams maintain project stability through version history, branching, and conflict resolution features.
